Refine the Deal Builder UI again so it reflects the actual West Hills Capital operating workflow.

Important business context:
- This is an internal tool used during live phone calls
- FedEx Hold is the standard and preferred delivery workflow
- Residential delivery is generally not the normal path
- The reason is operational and security-based: missed signatures and porch theft risk
- In limited cases, shipment may go to a customer’s business when they will be present and regularly receive deliveries
- The FedEx location list is highly useful operationally and should remain central to the workflow

Do not redesign this as a customer-facing experience. Optimize for operator speed, clarity, and safe fulfillment.

Preserve all existing working business logic unless explicitly changed below.

GOAL:
Make the Deal Builder reflect that FedEx Hold is the normal path, improve delivery UX, populate location hours, format product pricing correctly, and further sharpen the page for internal use.

--------------------------------------------------
1. DELIVERY OPTIONS MUST MATCH REAL OPERATING PRACTICE
--------------------------------------------------

FedEx Hold should remain the default and primary delivery workflow for CASH deals.

Required changes:
- Keep FedEx Hold preselected by default for CASH deals
- Visually make FedEx Hold the primary operational path
- Do not treat Home Delivery as an equal default peer

Because residential delivery is rarely used, update the secondary option so it better reflects reality.

If practical without breaking logic:
- Rename “Home Delivery” to something more operationally accurate, such as:
  - Business / Alternate Delivery
  OR
  - Alternate Address Delivery

Use the better label if implementation is straightforward.
If renaming would create avoidable downstream issues, keep the current label for now, but structure the UI so it is visually secondary.

--------------------------------------------------
2. FEDEX HOLD SHOULD FEEL LIKE THE STANDARD WORKFLOW
--------------------------------------------------

For CASH deals, the Delivery section should be optimized around FedEx Hold.

When FedEx Hold is selected:
- Show ZIP search
- Show Search button
- Show location results / selected location card
- Make this flow feel native and central, not like an exception path

Keep the FedEx location search prominent because it is operationally valuable.

Do not bury or minimize this feature.

--------------------------------------------------
3. POPULATE AND DISPLAY FEDEX LOCATION HOURS
--------------------------------------------------

Current issue:
- Office / location hours are not being populated in the UI

Fix:
- When a FedEx Hold location is selected, populate and display its hours if available from the location data source
- Show hours in the selected location display area
- If hours are unavailable, show a clean fallback such as:
  “Hours unavailable”

Do not leave the hours area blank if the UI includes it.

--------------------------------------------------
4. IMPROVE SELECTED FEDEX LOCATION DISPLAY
--------------------------------------------------

Once a FedEx Hold location is selected, display it as a clear selected card.

The selected card should show:
- FedEx Office / Ship Center name
- Full address
- Hours
- Clear visual indication that this is the selected location

Goal:
The operator should be able to quickly confirm:
- where the package is going
- whether the location is correct
- whether hours are available

--------------------------------------------------

--------------------------------------------------

Current issue:
- Product unit prices are not consistently formatted like other financial figures

Fix:
- Format all product Unit Price values as currency
- Use the same style as other financial numbers on the page
- Include dollar sign and standard decimal precision

Example:
$5,082.22
$89.16

Apply this consistently across:
- product rows
- any related calculated financial display where needed
